VelocityModelBasedLineInserter    Property List     v1.6.1

In the header of the main templates* for input artefacts* of this plugin the following artefact properties can be defined (italic properties are specific to this generator):

1. TargetDir – Folder in your file system where to write the generated content. If not defined, output will only be written in the plugin’s output directory. Use “<applicationRootDir>” (without quotes) to address the directory where the start script startMOGLiCodeCreator is located.

2. TargetFileName – Name of file in which the generated content is inserted.

3. CreateNew –  If not defined, default ‘false’ is used. This should be the standard case for insert jobs. If the target folder or the target file does not exist, an error is reported. If ‘true’, the existing target file is deleted and a non-existing one created (as well as its non-existing target folder). This way, the VelocityModelBasedInserter also can also be used as a “model based file maker”! Note, that this property cannot be used in combination with Insert or Replace properties.

4. NameOfValidModel – Determines that the template is only applied if the name of the current model (provided by the StandardModelProvider) is equal to the value of this template property. If not provided, the template will be never applied.

5. OutputEncodingFormat – Encoding applied when generating the artefact’s output file(s). Default is UTF-8. Since 1.1.0

6. SkipGeneration – If ‘true’, code supposed to be created by this template is not generated. Default is false. Can be used to suppress the generation of a specific artefact. The following use cases are applicable: a)  @SkipGeneration $isDatabaseRelevant, b) @SkipGeneration NOT $isDatabaseRelevant, c) @SkipGeneration $type == DomainObject, d) @SkipGeneration $type != DatabaseObject. Since 1.2.0 – c, d Since 1.6.0

7. ReplaceToNumberSign – Any string that will be replaced by the # symbol after Velocity has merged model and template and before the output file is created. Since Velocity interprets all number signs and its following chars as comment, these comments are removed during the Velocity merge of model and template. Thus, no number sign can get through this merge. As a workaround to get number sign in the final output, a number sign placeholder can be defined by this artefact property. Since 1.6.0

8. CheckMissingMetaInfos – If ‘false’, metainfo used in template files but not defined in the model data are ignored. If ‘true’, an error report is created in case a metainfo could not be resolved. Default is true. Since 1.6.1

9. ReplaceStart – Defines a single line text after which the lines will be replaced by the generated content. This attributes requires the additional attribute “ReplaceEnd”. Note, that this property cannot be used in combination with Insert or CreateNew properties.

10. ReplaceEnd – Defines a single line text before which the lines will be replaced by the generated content. This attributes requires the additional attribute “ReplaceStart”. Note, that this property cannot be used in combination with Insert or CreateNew properties.

11. InsertBelow – Defines a sequence of chars (the marker) to be found in a single line within the target file after which the generated content is injected into the target file’s content. If the marker is found more than once, the insertion takes please for each occurrence of the marker, but only if the generated content is not already contained below the marker. Note, that this property cannot be used in combination with Insert, Replace or CreateNew properties.

12. InsertAbove – Defines a sequence of chars (the marker) to be found in a single line within the target file. The generated content is injected into the target file in front of this marker. If the marker is found more than once, the insertion takes please for each occurrence of the marker, but only if the generated content is not already contained above the marker. Note, that this property cannot be used in combination with Insert, Replace or CreateNew properties.

 

* An input artefact is a subdirectory in <applicationRootDir>/input/VelocityModelBasedLineInserter that contains a main template. A main template’s name contains ‘Main’. A main template may call other templates. All templates have the file extension tpl. Input artefacts of the VelocityModelBasedLineInserter must have at least one main template. If more than one main template exist, they must refer to the same target file. The alphabetical order of input artefact subdirectories and of the main templates determines the order of generation events!
